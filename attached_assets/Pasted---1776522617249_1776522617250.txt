화면들 꼼꼼히 봤습니다. 완성도가 꽤 높은 앱이네요 — 단순 “종목 추천” 수준을 넘어서 매수그물·분할익절·박스권·백테스트까지 한 흐름으로 엮은 구조는 확실히 차별적입니다. 그래서 피드백도 그만큼 구체적으로 드릴게요. 잘한 점은 짧게, 개선점은 우선순위 순으로 정리합니다.
잘 만든 점
	•	진입/익절/박스권 탭 구조가 스윙 트레이더의 의사결정 순서와 일치합니다. “종목 찍어줌” 식 앱들과 결정적으로 다른 지점.
	•	5% 매수그물 + 3·5·8·15 분할익절 + MA5·MA20 -2% 손절까지 한 종목 안에서 닫혀 있는 규칙으로 제시된 점. 사용자가 재량으로 흔들릴 여지를 줄여줍니다.
	•	백테스트 탭에 왕복 비용(0.30%)과 MDD를 같이 노출한 건 아마추어 앱에서 거의 안 보이는 디테일입니다. “기간 단순등락 +522% vs 전략 순수익 +7.5%“를 같이 보여주는 것도 정직해서 좋습니다.
	•	박스권/재무/기술/리스크/요일/뉴스/이스라엘까지 탭을 쪼갠 정보 아키텍처. 한 화면에 다 때려넣지 않은 게 핵심.
	•	관심종목 리스트의 S1/PP/R1 피벗 슬라이더는 한 줄에 현재 위치를 직관적으로 보여줍니다. 계속 가져가세요.
개선이 필요한 점 (우선순위 순)
1. 논리적 모순 — 이게 가장 크리티컬합니다
샌디스크 한 종목 안에서 서로 충돌하는 신호가 동시에 떠 있어요.
	•	상단 배지: “고점권 · 저평가 · 1차 진입 -5%”
	•	기술·진단: RSI 78.2, MA20 +21.5%, MA60 +39.9%, 52주 고점 95.3% → 97/100 극과열 · “신규 진입 자제”
	•	그런데 같은 화면에 ”-5% 하락 시 추가매수” 매수그물이 그대로 활성화되어 있음
“극과열 — 신규 진입 자제”라고 판단했으면, 매수그물 1차 진입가는 최소한 회색 처리/비활성화되거나 경고 오버레이가 떠야 합니다. 지금은 두 모듈이 서로 대화하지 않고 각자 소리치고 있어요. 스윙 앱에서 이 정합성이 깨지면 사용자는 결국 본인이 원하는 화면만 골라보게 되고, 앱의 판단을 신뢰하지 않게 됩니다.
해결책: 최상단에 통합 상태 1줄을 두세요. 예: 상태: 극과열 — 신규 진입 보류 / 기존 보유자 분할 익절 고려. 그리고 이 상태에 따라 하위 모듈(매수그물, 익절)의 활성/비활성을 연동.
2. 원화 환산 표시가 과도합니다
₩30만, ₩50만, ₩7만, ₩135만… 미국 주식을 원화 주가로 1차 표시하고 달러를 작게 두는 건 직관을 떨어뜨립니다. NVDA를 “₩30만짜리 주식”으로 인식하는 건 의사결정에 방해가 돼요. 특히 매수그물 단가도 전부 ₩로 찍혀 있어서 실제 $로 주문할 때 머릿속 변환이 한 번 더 들어갑니다.
해결책: 미국장은 $ 기준가를 메인, ₩을 보조로. 토글로 반전 가능하게. 매수그물/익절 단가는 반드시 달러도 병기.
3. 데이터 품질 버그
	•	리스크 탭의 “전월 공매도 잔고: 1970-04-09T14:23:34” — 유닉스 에포크 근처 값이 그대로 노출. null 처리 누락입니다. 이거 하나로 앱 전체 신뢰도가 훅 떨어져요.
	•	“추세: 데이터 없음”인데 “공매도 위험 지수 21 / 보통”으로 확정값이 나오는 것도 모순.
	•	애널리스트 목표가 **-0.9%**인데 재무 평가는 “저평가”. 목표가 대비 상단에 붙어있으면 “저평가”라는 단어는 재고가 필요합니다. → “실적 기반 저평가 (주가는 목표가 근접)” 같은 뉘앙스 분리 필요.
4. 매수그물 전략의 본질적 문제 (투자자에게 해줘야 할 이야기)
이건 UI가 아니라 상품 설계 문제라 조심스럽게 드립니다. -5%, -10%, -15%, -20%마다 비중을 늘려가며 사는 피라미딩 물타기 구조는, 추세가 꺾인 종목에서는 손실을 가속시킵니다. 6개월 백테스트 +7.5%, 적중률 100%는 상승장 + 종목이 살아남았을 때의 결과이고, MDD -14.1%가 이미 그 경고를 주고 있습니다.
	•	종목이 -25% 이상 빠지는 경우의 시뮬레이션이 빠져 있음
	•	“T+5 타임스톱”이 있는 건 좋은데, 타임스톱이 걸리면 어떤 손실로 빠져나오는지 실제 backtest 숫자로 보여줘야 함
	•	백테스트 구간을 1개월/3개월/6개월/1년만 제공하는 것도 위험합니다. 최소 3년 또는 직전 약세장 구간(2022년 등)을 포함해야 전략의 진짜 얼굴이 나옵니다.
해결책: 백테스트에 “약세장 시나리오” 탭 추가 (2022 하락장, 개별 종목 -30% 구간). 그리고 매수그물 활성 조건에 “MA200 상단 + RSI < 70” 같은 필터를 달아서, 극과열 상태에서는 전략 자체가 트리거되지 않도록.
5. “AI 분석” 버튼의 역할이 모호함
우측 상단 “AI 분석” 버튼이 뭘 하는지 화면만 봐서는 안 보입니다. 이미 모든 탭이 분석인데 별도 AI 버튼이 있으면 — 나머지는 AI가 아니라는 뜻인가? 라는 의문이 생겨요. 차라리 “AI 요약”으로 이름 바꾸고, 모든 탭의 결론을 3줄로 요약해주는 역할로 특화시키는 게 낫습니다.
6. “이스라엘” 탭의 애매한 위치
현재는 지정학 리스크 중 이스라엘만 튀어나와 있어서 어색합니다. “지정학·공급망” 같은 상위 탭 하나 두고 그 안에 이스라엘/중국/대만/러시아 노출도를 같이 묶는 게 자연스럽습니다. 종목에 따라 해당 없으면 접어두면 되고요.
7. 사소하지만 거슬리는 것들
	•	관심종목 리스트에 “저점 기회 2, 세력 진입 0, 고점 주의 15”가 요약으로 떠 있는데 — 23개 중 15개가 고점주의면 이건 리스트 자체가 상단에 편중된 신호입니다. “지금 신규 매수 적합 종목이 거의 없음”이라는 메시지를 상단에 띄워주면 오히려 신뢰가 쌓입니다.
	•	NVDA/GOOGL 모두 고점권 + 슬라이더는 상단 붙음 + 등락률은 +1.68%로 동일 — 같은 % 숫자가 두 종목에 우연히 똑같이 찍힌 건지, 하드코딩된 플레이스홀더인지 의심이 듭니다. 확인 필요.
	•	요일별 유의사항(“화요일: NVDA 흐름 참고”, “금요일: 고점 매도 기회”)은 근거 없는 점성술처럼 보일 수 있습니다. 과거 데이터 통계로 근거를 보여주거나, 아예 빼는 게 낫습니다.
	•	탭 7~8개를 가로스크롤로 넣은 건 모바일에서 끝 탭(백테스트/이스라엘)이 잘 안 보입니다. 중요도 낮은 건 “더보기”로 접기.
한 줄로 정리
“각 모듈은 잘 만들었는데, 모듈 간 대화가 없다” — 이게 현재 이 앱의 가장 큰 숙제입니다. 모듈들을 관통하는 단일 상태 머신(관망/진입/보유/분할익절/청산)을 만들어서 모든 탭이 그 상태를 존중하도록 리팩터링하면, 지금의 정보량만으로도 체급이 한 단계 올라갑니다.
